# docs/

프로젝트 문서 폴더. 번호 접두어로 워크플로우 순서 정렬.

## 폴더 구조

| 폴더 | 용도 |
|------|------|
| `00_backlog/` | 백로그 — 작업 대기/진행/완료 관리 |
| `10_planning/` | 기획서, 요구사항 |
| `20_spec/` | 스펙, 아키텍처, 설계 문서 |
| `50_review/` | 리뷰 기록, 결정 사항 |
| `90_logs/` | 운영 로그, 릴리즈 노트 |

## 네이밍 규칙
- 폴더 앞에 두 자리 숫자 — 워크플로우 순서와 정렬 일치
- 번호 사이 여백(30~40, 60~80)은 의도적으로 비워둠 — 추후 삽입 여지
- 새 폴더 추가 시 이 README에 등록
