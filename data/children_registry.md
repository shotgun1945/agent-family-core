# 자식 프로젝트 레지스트리

Family Core가 관리하는 자식 프로젝트 목록.  
경로: `../{username}_children/`

---

## 운영 원칙

- 자식 프로젝트 파일은 유저가 명시적으로 요청할 때만 읽는다 (토큰 절약)
- 각 프로젝트의 `CLAUDE.md`를 먼저 읽고 컨텍스트를 파악한다
- 자식 프로젝트에 변경사항이 생기면 해당 프로젝트의 파일에 저장한다

---

## 자식 생성 체크리스트

`create-child` 스킬 실행 시 아래 항목을 수행한다:

1. `data/children_manifest.md` 확인 — 전파할 스킬·문서 목록 파악
2. `../{username}_children/{child-name}/` 폴더 생성
3. `CLAUDE.md` 작성 — 목적, 구조, `## 부모 연동` 섹션 포함
4. 매니페스트 `자식 로컬 복사 매핑` 기준으로 스킬·문서 복사
5. 백로그 파일 생성
6. 이 레지스트리(`data/children_registry.md`)에 항목 추가
