# 페르소나 파일 갱신 기준

## 갱신 원칙
- 유저가 명시적으로 언급한 정보만 저장한다
- 추측이나 추론은 저장하지 않는다
- 갱신 전 기존 내용과 충돌 여부를 확인한다

## 파일별 갱신 조건

### `profile.md`
저장 조건:
- 직업, 역할, 소속이 언급될 때
- 새 프로젝트가 시작되거나 기존 프로젝트가 완료될 때
- 목표나 우선순위가 바뀔 때

### `preferences.md`
저장 조건:
- 응답 방식에 대한 피드백을 줄 때 ("더 짧게", "코드만 보여줘")
- 도구나 언어 선호를 언급할 때
- 금기 사항을 명시할 때

### `personality.md`
저장 조건:
- 성격, 작업 습관, 동기에 대해 직접 언급할 때

### `assistant_persona.md`
저장 조건:
- AI의 이름, 말투, 역할에 대한 변경을 요청할 때

## 저장 금지
- 현재 대화의 일시적 맥락 (특정 작업 세부사항)
- 부정적 판단이나 평가성 정보
- 코드 패턴, 아키텍처 등 코드에서 직접 읽을 수 있는 정보
